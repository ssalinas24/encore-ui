Design Patterns within Encore

  • When should you use modals instead of a new page?
    • According to Bootstrap, "Modals are streamlined, but flexible, dialog prompts with the minimum required functionality and smart defaults."
    • In Encore, you should use modals for confirmation messaging. For notifications you will use rxNotification, which is the static subdirective of rxNotify. See the example below which uses type="error". type="error" can be replaced with the appropriate message response style.
    • You also can use modals to edit content in lieu of having an edit view. This design pattern is not ideal. An Edit View will need to be considered as a pattern in future iterations.
  • Using the rxModalForm subdirective will automatically provide you with a title, subtitle, submit and cancel text. You can use rxForm elements within rxModalForm.
  • For real world examples, take a look at the ng-template blocks triggered from server action links under Encore Cloud.
  • If you need to deviate from the title / footer pattern at all, you'll need to take a look at the rxModalForm directive template markup and build upon that.
